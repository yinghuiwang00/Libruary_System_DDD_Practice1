> 本文档说明项目中限界上下文之间的集成方式——基于 Kafka 事件驱动的异步通信

## 集成方式：双发模式

每个上下文的事件发布器采用**双发模式 (Double Publishing)**：

1. **本地同步发布**：通过 Spring `ApplicationEventPublisher`，保证本上下文内的事件处理必定成功
2. **异步 Kafka 发布**：通过 `KafkaTemplate`，跨上下文通信，Kafka 不可用时优雅降级

```
领域服务 → *DomainEventPublisher.publish(event)
              ├── ① localPublisher.publish(event)   // 同步，必成功
              └── ② kafkaTemplate.send(topic, event) // 异步，可选
```

## 上下文角色

| 上下文 | 角色 | 发布者 | 消费者 | 说明 |
|--------|------|--------|--------|------|
| Catalog | 发布者 | 1 | 0 | 只发布事件，不消费其他上下文 |
| Inventory | 发布+消费 | 1 | 2 | 消费 Catalog 和 Circulation 事件 |
| Circulation | 发布+消费 | 1 | 1 | 消费 Patron 事件 |
| Patron | 发布+消费 | 1 | 2 | 消费 Circulation 和 Payment 事件 |
| Payment | 发布+消费 | 1 | 1 | 消费 Circulation 事件 |
| Analytics | 发布+消费 | 1 | 2 | 消费 Catalog 和 Inventory 事件 |
| Notification | 纯消费 | 0 | 4 | 消费 4 个上下文的事件 |

## 集成事件流详解

### 流程 1：新书入库 (Catalog → Inventory)

```
Catalog                        Inventory
  │                               │
  │── BookCreatedEvent ──────────→│
  │   (bookId, isbn, title)       │
  │                               ├── BookCreatedEventHandler
  │                               │   → 创建 CopyInventory 记录
  │                               │   → 为新书创建初始副本
```

**触发**: 图书管理员在编目上下文创建新书
**效果**: 库存上下文自动在默认图书馆创建库存记录

### 流程 2：借书 (Circulation → Patron, Inventory, Notification, Payment)

```
Circulation          Patron           Inventory        Notification       Payment
    │                   │                  │                │                │
    │─ BookBorrowedEvent ─→│                  │                │                │
    │  (patronId, copyId)  │                  │                │                │
    │                   ├── recordLoan()    │                │                │
    │                   │   增加借阅计数     │                │                │
    │                   │                  │                │                │
    │─ BookBorrowedEvent ──────────────────→│                │                │
    │  (copyId, bookId)  │                  │                │                │
    │                   │                  ├── markAsBorrowed()              │
    │                   │                  │   更新副本状态   │                │
    │                   │                  │                │                │
    │─ BookBorrowedEvent ─────────────────────────────────→│                │
    │  (patronId, bookId)│                  │                │                │
    │                   │                  │                ├── 创建借阅通知  │
```

### 流程 3：还书 (Circulation → Patron, Inventory, Notification)

```
Circulation          Patron           Inventory        Notification
    │                   │                  │                │
    │─ BookReturnedEvent →│                  │                │
    │  (patronId, copyId) │                  │                │
    │                   ├── recordReturn()  │                │
    │                   │   减少借阅计数     │                │
    │                   │   检查逾期并计算罚款│                │
    │                   │                  │                │
    │─ BookReturnedEvent ──────────────────→│                │
    │                   │                  ├── markAsReturned()│
    │                   │                  │                │
    │─ BookReturnedEvent ─────────────────────────────────→│
    │                   │                  │  ├── 创建归还通知 │
```

### 流程 4：罚金→支付链 (Circulation → Payment → Patron → Notification)

```
Circulation      Payment           Patron          Notification
    │               │                 │                 │
    │─ FineIncurredEvent ─→│           │                 │
    │  (patronId, amount)  │           │                 │
    │               ├── 创建 Payment   │                 │
    │               │   记录            │                 │
    │               │                 │                 │
    │─ FineIncurredEvent ──────────────→│                 │
    │               │  ├── addFine()   │                 │
    │               │  │  增加罚款余额  │                 │
    │               │                 │                 │
    │               │                 │                 │
    │               │─ PaymentCompletedEvent ──→│         │
    │               │  (patronId, amount)       │         │
    │               │                 ├── payFine()      │
    │               │                 │  减少罚款余额     │
    │               │                 │                 │
    │               │─ PaymentCompletedEvent ────────────→│
    │               │                 │  ├── 创建支付确认通知
```

### 流程 5：读者暂停 (Patron → Circulation, Notification)

```
Patron           Circulation       Notification
  │                  │                 │
  │─ PatronSuspendedEvent ──→│         │
  │  (patronId, reason)      │         │
  │                  ├── 记录暂停事件   │
  │                  │  影响借阅权限    │
  │                  │                 │
  │─ PatronSuspendedEvent ────────────→│
  │                  │  ├── 创建状态变更通知
```

### 流程 6：预约 (Circulation → Notification)

```
Circulation          Notification
    │                    │
    │─ HoldPlacedEvent ──→│
    │  (patronId, bookId) │
    │                    ├── 创建预约通知
    │                    │
    │─ HoldFulfilledEvent →│
    │  (patronId, copyId) │
    │                    ├── 创建可取通知
    │                    │
    │─ OverdueNoticeEvent →│
    │  (patronId, loanId) │
    │                    ├── 创建逾期通知
```

### 流程 7：低库存预警 (Inventory → Notification, Analytics)

```
Inventory          Notification      Analytics
    │                  │                │
    │─ LowStockAlertEvent ──→│          │
    │  (bookId, available)   │          │
    │                  ├── 创建预警通知  │
    │                  │  (给 LIBRARIAN) │
    │                  │                │
    │─ LowStockAlertEvent ──────────────→│
    │                  │  ├── 记录统计    │
```

### 流程 8：统计分析 (Catalog → Analytics)

```
Catalog            Analytics
    │                │
    │─ BookCreatedEvent ──→│
    │  (bookId, isbn, title)│
    │                ├── BookCreatedAnalyticsHandler
    │                │   → 记录新书统计
```

## Kafka Topic 配置

| Topic 名称 | 发布者 | 消费者数量 | 用途 |
|-----------|--------|----------|------|
| library.catalog.events | Catalog | 2 (Inventory, Analytics) | 图书状态变更 |
| library.circulation.events | Circulation | 4 (Patron, Inventory, Payment, Notification) | 借阅/归还/罚金/预约事件 |
| library.patron.events | Patron | 2 (Circulation, Notification) | 读者状态变更 |
| library.inventory.events | Inventory | 2 (Notification, Analytics) | 库存/副本变更 |
| library.payment.events | Payment | 2 (Patron, Notification) | 支付/退款事件 |
| library.analytics.events | Analytics | 0 | 报表事件（暂无消费者） |

## 消费者 Group ID 命名规范

```
library.<消费模块>.consumer.<来源模块>
```

示例：
- `library.inventory.consumer.catalog` — 库存上下文消费编目事件
- `library.patron.consumer.circulation` — 读者上下文消费流通事件
- `library.notification.consumer.payment` — 通知上下文消费支付事件

## 事件消费的容错设计

| 策略 | 实现 |
|------|------|
| Kafka 不可用降级 | `ObjectProvider<KafkaTemplate>` 可选注入 |
| 发送失败不影响业务 | `try-catch` + `whenComplete` |
| 未知事件类型忽略 | `default -> log.debug("Ignoring unknown event type")` |
| 消费异常不重抛 | `catch (Exception e) { log.error(...) }` 防止 poison pill |
| 异步断言 | Awaitility `await().untilAsserted()` 等待消费完成 |
| 测试隔离 | `@DirtiesContext` + `@Before` 清理所有表 |
