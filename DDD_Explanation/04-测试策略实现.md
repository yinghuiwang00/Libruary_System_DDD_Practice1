## E2E 测试架构

### library-e2e-test (JUnit 5)

- **特点**: 直接验证 Kafka 消费端的跨上下文集成
- **模式**: `@SpringBootTest` + `@EmbeddedKafka` + `KafkaTemplate.send()` + `await().untilAsserted()`
- **基础设施**: `BaseEndToEndTest` 提供 Kafka 生产者创建 + DB 清理 + JSON 构建
- **启动类**: `IntegrationTestApplication` 扫描全部 7 个上下文
- **配置类**: `EmbeddedKafkaConfig` (5个Topic) + `IntegrationTestBeansConfig`

| 测试类 | 覆盖场景 | 验证的跨上下文通信 |
|--------|---------|-----------------|
| BorrowBookEndToEndTest | UC-1 借书 | Circulation→Patron, Circulation→Notification |
| ReturnBookEndToEndTest | UC-2 还书 | Circulation→Patron, Circulation→Notification |
| HoldBookEndToEndTest | UC-3 预约 | Circulation→Notification |
| FinePaymentEndToEndTest | UC-4/5 罚金/支付 | Circulation→Patron→Payment→Notification |
| NewBookEndToEndTest | UC-6 新书上架 | Catalog→Inventory |
| PatronSuspensionEndToEndTest | UC-7 读者暂停 | Patron→Notification |
| LowStockAlertEndToEndTest | UC-8 低库存预警 | Inventory→Notification |

### library-integration-test (Cucumber BDD)

- **特点**: 业务可读的 Given/When/Then 跨上下文验证
- **模式**: `@CucumberContextConfiguration` + `@EmbeddedKafka` + Step 定义类
- **基础设施**: `CucumberSpringConfig` + `E2EScenarioState` + `SharedSteps`
- **Gherkin 示例**:
  ```gherkin
  Feature: Borrow Book Cross-Context Integration

    Scenario: Borrow book updates patron loan count and creates notification
      Given a patron "John Doe" with email "john.borrow@test.com" exists
      When a BookBorrowedEvent is published for that patron with loan "loan-001", copy "copy-001", book "book-001"
      Then the patron's current loan count should be 1
      And a notification should exist for the patron
  ```

| Feature 文件 | 覆盖场景 | Step 定义类 |
|-------------|---------|-----------|
| borrow-book.feature | 借书跨上下文 | BorrowBookSteps |
| return-book.feature | 还书跨上下文 | ReturnBookSteps |
| hold-book.feature | 预约跨上下文 | HoldBookSteps |
| fine-payment.feature | 罚金/支付跨上下文 | FinePaymentSteps |
| new-book.feature | 新书上架跨上下文 | NewBookSteps |
| patron-suspension.feature | 读者暂停跨上下文 | PatronSuspensionSteps |
| low-stock-alert.feature | 低库存预警跨上下文 | LowStockAlertSteps |

## 测试基础设施

| 组件 | 用途 |
|------|------|
| H2 (PostgreSQL 兼容模式) | 内存数据库，免外部依赖 |
| EmbeddedKafka | 内存 Kafka Broker，5 个 Topic |
| Awaitility | 异步断言，等待 Kafka 消费完成 |
| `@DirtiesContext` | 每个 Scenario/TestClass 重置 Spring 上下文 |
| `@Before` 清理 SQL | 清理所有表（逆 FK 依赖顺序） |
| E2EScenarioState | BDD 测试共享状态（patronId, bookId 等） |
| BaseEndToEndTest | JUnit5 E2E 基类（Kafka 生产者 + DB 清理 + JSON 构建） |

## 运行命令

```bash
# 运行全部测试
mvn clean test

# 运行单个模块测试
cd library-circulation && mvn test

# 运行 E2E 测试
cd library-e2e-test && mvn test
cd library-integration-test && mvn test

# 跳过测试构建
mvn clean install -DskipTests
```
