# 聚合根模式实现说明

> 本文档说明项目中聚合根的实际实现模式

## 聚合根统一模式

所有聚合根遵循以下模式：

```java
@Entity
public class Patron {
    @EmbeddedId
    private PatronId id;           // 自定义 ID 类型（共享值对象）

    @Version
    private Long version;          // 乐观锁

    @Column(name = "created_at")
    private LocalDateTime createdAt;  // 审计字段

    @Column(name = "updated_at")
    private LocalDateTime updatedAt;  // 审计字段

    // 静态工厂方法（不暴露构造函数）
    public static Patron create(String firstName, String lastName,
                                String email, PatronType type) { ... }

    // 业务方法（不暴露 setter）
    public void recordLoan() { ... }
    public void recordReturn() { ... }
    public void addFine(BigDecimal amount) { ... }
    public void suspend(String reason) { ... }
    public void reactivate(String reason) { ... }
    public void terminate(String reason) { ... }
}
```

## ID 值对象体系

所有 ID 继承自 `AggregateId` 基类：

```java
@Embeddable
@MappedSuperclass
public abstract class AggregateId implements Serializable, Comparable<AggregateId> {
    @Column(name = "id", nullable = false)
    private String value;

    protected AggregateId() {}
    protected AggregateId(String value) {
        this.value = Objects.requireNonNull(value, "ID value must not be null");
    }

    public static String generateUUID() {
        return UUID.randomUUID().toString();
    }

    // equals(), hashCode(), toString(), compareTo()
}
```

## 乐观锁

所有聚合根使用 `@Version` 注解实现乐观锁。当两个请求同时修改同一聚合时，后到的请求会抛出 `OptimisticLockingFailureException`。

## 审计字段

所有聚合根包含 `createdAt` 和 `updatedAt` 审计字段，通过 JPA `@CreatedDate` / `@LastModifiedDate` 自动填充（由各模块的 `JpaConfig` 中 `@EnableJpaAuditing` 启用）。
