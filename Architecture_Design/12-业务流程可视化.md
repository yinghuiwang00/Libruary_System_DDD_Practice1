### 2.2 借书业务规则

```mermaid
graph TD
    Start[开始借书] --> ValidatePatron[验证读者资格]
    ValidatePatron -->|不合格| Error1[返回错误:读者不具备借阅资格]
    
    ValidatePatron -->|合格| CheckLoans[检查当前借阅数量]
    CheckLoans -->|达到上限| Error2[返回错误:借阅数量已满]
    
    CheckLoans -->|未达上限| CheckOverdue[检查是否有逾期]
    CheckOverdue -->|有逾期| Error3[返回错误:存在逾期图书]
    
    CheckOverdue -->|无逾期| CheckDuplicate[检查是否重复借阅]
    CheckDuplicate -->|已借阅| Error4[返回错误:已借阅该书]
    
    CheckDuplicate -->|未借阅| CheckHolds[检查是否有等待预约]
    CheckHolds -->|有预约且非首位| Error5[返回错误:该书有等待预约]
    
    CheckHolds -->|无预约或首位| CreateLoan[创建借阅记录]
    CreateLoan --> UpdateCopy[更新副本状态]
    UpdateCopy --> UpdatePatron[更新读者借阅数]
    UpdatePatron --> SendNotification[发送借阅通知]
    SendNotification --> Success[借阅成功]
    
    Error1 --> End[结束]
    Error2 --> End
    Error3 --> End
    Error4 --> End
    Error5 --> End
    Success --> End
```

## 11. 异常处理流程

### 11.1 错误处理流程

```mermaid
graph TD
    Start[操作开始] --> Try[尝试执行业务逻辑]
    Try -->|成功| Success[操作成功]
    Try -->|失败| Catch[捕获异常]
    
    Catch --> DomainException{领域异常}
    DomainException --> Log[记录日志]
    Log --> ErrorResponse[返回错误响应]
    
    Catch --> ValidationException{验证异常}
    ValidationException --> Log
    Log --> ErrorResponse
    
    Catch --> ResourceException{资源异常}
    ResourceException --> Log
    Log --> ErrorResponse
    
    Catch --> SystemException{系统异常}
    SystemException --> Log
    Log --> ErrorResponse
    
    ErrorResponse --> End[操作结束]
    Success --> End
```

```mermaid
sequenceDiagram

    Local-->>Local: 本地同步处理

    Kafka->>NotifCons: @KafkaListener 消费

    NotifCons->>NotifCons: notificationRepository.save()
    
    Kafka->>InvCons: @KafkaListener 消费
```

### 10.4 双发模式流程

```mermaid
flowchart LR
    A["领域服务操作"] --> B["*DomainEventPublisher<br/>.publish(event)"]
```
